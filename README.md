## 🛠️ Installation & Setup

> [!NOTE]
> Follow these steps to compile and run the bot from source code.

<br/>

### 1️⃣ Prerequisites

<table>
<tr>
<td>

| Requirement | Version | Download |
|:------------|:--------|:---------|
| **Java (JDK)** | 17 or newer | [Adoptium Temurin](https://adoptium.net/) |
| **Apache Maven** | 3.8+ | [Download Maven](https://maven.apache.org/install.html) |

</td>
</tr>
</table>

<details>
<summary><b>💡 Windows Users: Adding Java & Maven to PATH</b></summary>
<br/>

1. Press <kbd>Win</kbd> + <kbd>R</kbd>, type `sysdm.cpl`, and press <kbd>Enter</kbd>
2. Go to **Advanced** → **Environment Variables**
3. Under **System variables**, select `Path` and click **Edit**
4. Add the `bin` directories of your Java and Maven installations:
   ```
   C:\Program Files\Eclipse Adoptium\jdk-17\bin
   C:\apache-maven-3.9.9\bin
   ```
5. Click **OK** and restart your terminal
6. Verify with:
   ```sh
   java -version
   mvn -version
   ```

</details>

<br/>

### 2️⃣ Compilation

```sh
# Clone the repository
git clone https://github.com/Shederator/wosbot.git
cd wosbot

# Build the project
mvn clean install package
```

> [!TIP]
> Once completed, the compiled `.jar` file will be in the `wos-hmi/target` directory (e.g., `wos-bot-1.7.0.jar`).

<br/>

### 3️⃣ Running the Bot

```sh
# Navigate to the target directory
cd wos-hmi/target

# Run the compiled jar
java -jar wos-bot-1.7.0.jar
```

> [!IMPORTANT]
> We highly recommend running via command line to access real-time logs for easier debugging. Double-clicking the `.jar` works, but console logs won't be visible.

<div align="center">
  <i>The bot interfaces with your Android emulator. Officially supported emulators:</i>

  ![MuMu](https://img.shields.io/badge/MuMu_Player-Supported-4CAF50?style=for-the-badge)
  ![MEmu](https://img.shields.io/badge/MEmu-Supported-4CAF50?style=for-the-badge)
  ![LDPlayer](https://img.shields.io/badge/LDPlayer_9-Supported-4CAF50?style=for-the-badge)

### Emulator Executable Paths

When prompted by the bot launcher, select the **command-line controller** (not the graphical app):

<details>
<summary><b>📂 MuMu Player</b></summary>

```
C:\Program Files\Netease\MuMuPlayerGlobal-12.0\shell\MuMuManager.exe
C:\Program Files\Netease\MuMuPlayer\nx_main\MuMuManager.exe
```
</details>

<details>
<summary><b>📂 MEmu</b></summary>

```
C:\Program Files\Microvirt\MEmu\memuc.exe
```
</details>

<details>
<summary><b>📂 LDPlayer 9</b></summary>

```
C:\LDPlayer\LDPlayer9\ldconsole.exe
```

> [!WARNING]
> You must manually enable ADB: `Settings → Other settings → ADB debugging → Enable local connection`
</details>
